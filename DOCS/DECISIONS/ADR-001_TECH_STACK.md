# ADR-001: 기술 스택 선택 — Phaser 3 + Vanilla JS

> **카테고리:** DECISIONS
> **최초 작성:** 2026-03-21
> **최종 갱신:** 2026-03-21
> **관련 기능:** 전체 프로젝트

**상태:** Accepted
**날짜:** 2026-03-21

---

## 맥락

모바일 환경을 대상으로 하는 2D 디펜스 게임을 제작해야 했다. 핵심 제약 조건은 다음과 같다.

- 별도 서버 없이 `file://` 프로토콜로 직접 실행 가능해야 한다
- 빌드 툴체인(webpack, vite 등) 없이 즉시 실행 가능해야 한다
- 외부 이미지/오디오 에셋 없이 코드만으로 시각 요소를 구현해야 한다
- 세로 모드 모바일 화면(390 × 844)에 맞게 스케일링이 자동으로 이루어져야 한다

---

## 결정

**Phaser 3 (CDN 방식)** + **Vanilla JavaScript (ES6 class 문법)** 조합을 선택한다.

- Phaser는 `<script src="https://cdn.jsdelivr.net/npm/phaser@3/dist/phaser.min.js">` 한 줄로 로드
- 모든 소스 파일은 `window.*` 전역 네임스페이스에 등록하여 import/export 없이 의존성을 관리
- 의존성 순서는 `index.html`의 `<script>` 태그 순서로 보장

---

## 이유

**Phaser 3를 선택한 이유:**

- `Phaser.GameObjects.Container`와 Graphics API로 외부 에셋 없이 도형 기반 렌더링이 가능하다
- `Phaser.Scale.FIT + CENTER_BOTH`로 임의 화면 크기에 자동 맞춤이 된다
- `time.addEvent`로 게임 루프 외부의 타이머(성 화살, 적 소환)를 씬 생명주기에 연동하여 관리할 수 있다
- 씬 시스템이 화면 전환 상태 관리를 명확하게 분리한다
- 물리 엔진을 비활성화하고 거리 계산을 직접 구현함으로써 불필요한 연산을 제거한다

**Vanilla JS + window.* 전역 방식을 선택한 이유:**

- 번들러 없이 `file://`로 직접 실행하려면 ES Module의 CORS 제한을 피해야 한다. `file://`에서 `type="module"` 스크립트는 대부분의 브라우저에서 CORS 오류로 로드가 거부된다
- `window.*` 전역 등록은 스크립트 로드 순서만 지키면 별도 설정 없이 의존성이 해결된다
- ES6 class 문법으로 상속(`Unit → AllyUnit → Hero`)을 명확하게 표현할 수 있다

---

## 검토한 대안

**React + Canvas 조합:**
- 빌드 툴체인이 필수로 요구되어 `file://` 직접 실행 요건을 충족하지 못한다
- 게임 루프와 React의 렌더링 사이클 간 충돌 가능성이 있다
- 기각.

**순수 Canvas API (Phaser 없음):**
- 씬 전환, 스케일링, 타이머 등을 직접 구현해야 하므로 개발 공수가 크게 증가한다
- 기각.

**PixiJS:**
- 렌더링에 특화되어 있으나 씬 관리, 타이머, 스케일링 기능이 Phaser에 비해 부족하다
- 별도 설치 없이 CDN 사용이 가능하지만, 게임 전용 기능 지원이 Phaser보다 적다
- 기각.

**ES Module + `file://`:**
- 대부분의 Chromium 기반 브라우저에서 `file://` 경로의 모듈 간 import가 CORS 정책으로 차단된다
- 로컬 서버 없이는 동작하지 않으므로 "직접 실행" 요건을 충족하지 못한다
- 기각.

---

## 결과

**긍정적 트레이드오프:**
- 설치 없이 `index.html` 파일 하나로 즉시 실행 가능
- 에셋 파일 관리 불필요 — 모든 시각 요소가 코드로 재현 가능

**부정적 트레이드오프:**
- 인터넷 연결이 없으면 CDN에서 Phaser를 로드할 수 없다 (오프라인 사용 시 로컬 복사 필요, `DOCS/OPERATIONS/SETUP_AND_RUN.md` 참조)
- `window.*` 전역 방식은 스크립트 로드 순서에 강하게 의존하므로, 파일 추가 시 `index.html`의 순서를 신중하게 관리해야 한다
- 타입 안전성이 없어 런타임 오류가 발생할 수 있다 (TypeScript 미사용)
