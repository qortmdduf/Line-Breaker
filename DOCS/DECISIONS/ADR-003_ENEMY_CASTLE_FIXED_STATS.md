# ADR-003: 적군 성 화살 수치 고정 — 플레이어 업그레이드 미반영

> **카테고리:** DECISIONS
> **최초 작성:** 2026-03-21
> **최종 갱신:** 2026-03-21
> **관련 기능:** BattleScene.js, Castle.js

**상태:** Accepted
**날짜:** 2026-03-21

---

## 맥락

플레이어는 `arrow_dmg`와 `arrow_spd` 업그레이드로 아군 성의 화살 데미지와 속도를 향상시킬 수 있다. 초기 구현에서 아군 성과 적군 성이 동일한 로직으로 화살 수치를 계산했기 때문에, 플레이어의 성 화살 업그레이드가 적군 성 화살 수치에도 동일하게 적용되는 버그가 있었다.

QA 12라운드 테스트 중 이 버그가 발견되었다. 수정 방향을 결정해야 했다.

---

## 결정

**적군 성 화살 수치는 항상 고정값을 사용한다.**

- 적군 성 화살 데미지: `GameConfig.ARROW_BASE_DAMAGE` (20, 고정)
- 적군 성 화살 속도: 300 px/s (고정)
- 플레이어의 `arrow_dmg`, `arrow_spd` 업그레이드는 아군 성에만 적용된다

BattleScene.js의 구현:

```javascript
// 아군 성: 업그레이드 적용
const arrowDmg = cfg.ARROW_BASE_DAMAGE + arrowDmgLevel * 10;
const arrowProjSpeed = 300 + arrowSpdLevel * 40;
this.allyCastle = new window.Castle(..., arrowDmg, arrowInterval, ...arrowProjSpeed...);

// 적군 성: 고정 수치 사용
this.enemyCastle = new window.Castle(..., cfg.ARROW_BASE_DAMAGE, arrowInterval, ...300...);
```

---

## 이유

성 화살 업그레이드의 목적은 플레이어가 방어력을 강화하여 더 어려운 스테이지에 대응할 수 있게 하는 것이다. 이 업그레이드가 적군 성에도 동일하게 적용된다면:

- 업그레이드를 많이 할수록 아군 유닛이 더 빠른 화살에 더 많은 데미지를 입게 된다
- 업그레이드가 오히려 전투를 불리하게 만드는 역설이 발생한다
- 플레이어는 업그레이드를 의도적으로 피해야 하는 비직관적인 상황이 만들어진다

적군 성의 강도는 스테이지 데이터(`enemyCastleHp`, `spawnInterval`)로 이미 조절되고 있다. 적군 성 화살 수치를 추가로 변동시킬 이유가 없다.

---

## 검토한 대안

**적군 성에도 별도 업그레이드 수치 적용 (별도 파라미터):**
- 적군 성 전용 화살 강화 파라미터를 스테이지 데이터에 추가하는 방안
- 스테이지별로 적군 성 화살을 다르게 설정할 수 있어 설계 유연성은 높아진다
- 그러나 현재 스테이지 설계에서 이를 활용할 계획이 없고, 데이터 복잡도만 증가한다
- 기각. 필요 시 추후 추가 가능.

**버그 수정 없이 유지:**
- 플레이어가 성 화살 업그레이드를 구매할수록 적군 성도 강해지는 것을 "의도된 밸런스"로 간주
- 이는 플레이어에게 직관적으로 설명될 수 없으며, 업그레이드 시스템의 가치를 훼손한다
- 기각.

---

## 결과

**긍정적 트레이드오프:**
- 성 화살 업그레이드의 효과가 플레이어에게 직관적이다 — 구매할수록 유리해진다
- 전투 난이도가 스테이지 데이터(HP, spawnInterval)와 업그레이드(유닛 스탯, 아군 화살)의 두 축으로 명확하게 분리된다

**부정적 트레이드오프:**
- 적군 성 화살 수치가 하드코딩되어 있으므로, 추후 스테이지별로 다른 적군 성 화살 수치가 필요하다면 구조 변경이 필요하다
